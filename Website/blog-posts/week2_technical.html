The Box Model

What is the difference between Margin, Border, and Padding?

  • Margin - is the padding outside of a border, between the border and the edge of the page. Adjusting this is useful to keep a clear edge around the page.
  • Border - is the visual border you see on the page, and can be easily styled for colour, thickness and format.
  • Padding - is the padding inside of a border, between the contents and the border. Adjusting this is useful to keep content text spaced from the edge of the content box.

What are the differences between relative, absolute, and fixed positioning?

  • Relative - this type of positioning automatically adjusts itself to itself, if you include a positioning attribute, e.g. bottom 10px where it will shift its position up 10px from where it would usually sit.
  • Absolute - this enables a page element to positioned exactly where you would like to to be, i.e. removing itself from the flow of elements on the page.
  • Fixed - this type of positioning is relative to the viewport (i.e. browser) and so the page stays where it is, stifling scrolling activities, and is pretty rare as a result.
  • Static - this is the default for every page element.

Elements and structure

What are the best practices associated with using classes vs. ids?

  • Classes - are not unique with the same class being usable on multiple elements. You can also use multiple classes on the same element.
  • Ids - are unique as each element can only have one ID (and each page can only have one element with that ID).

What does it mean to display inline code vs block code?

  • Inline code - is code that flows along with the text content, is typically shorter in length as these often address more minor amendments, e.g. setting a word to bold.
  • Block code - is code that takes up the full width available in its container (i.e. div or span etc), and so require formatting to ensure the contents fit within the container size.
