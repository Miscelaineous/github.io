Ruby vs Javascript

  • Ruby is an intepreted language, vs compiled language, which means that a separate compiler is not required, albeit at the cost of slightly slower performance.
  • Ruby is object-orientated, to the extent that every element is treated as an object.
  • Ruby is at the heart of the Ruby on Rails framework, albeit it doesn't have to run on this.
  • Files are saved as .rb so that text editors, such as Sublime, can recognise the text as Ruby code.
  • Interesting Fact: The name Ruby was chosen by Matz as it was a colleague's birthstone.
  • Inspirational web pagesbuilt on Ruby on Rails frameworks.
  • Javascript is a functional language that also heavily relies on the naming and defining of objects.
  • Its key strength is its ability to be run on a browser, with interactions, amendments and updates also possible, facilitating web-app interactivity. This makes Javascript incredibly dynamic in nature.
  • Interesting Fact: Javascript was developed under the name Mocha.
  • Inspirational web sitesbuilt using Javascript

Summary

Javascript has regained in popularity over the last few years as a rich and dynamic way to develop web apps, with enhanced interactivity features. Hence it's increasing use by web developers.

Ruby is a simple, easy to read and understand code. It focuses on slick tools and developers are increasingly adopting it. However, it's not necessarily a good place for beginners to start, particularly with the lack of syntax and syntactic sugar which make fundamental coding concepts and rules hard to learn.

A combination of both appears to be a good starting point for wannabe developers.

Looping

What is it?

Iteration is when a code is repeated to avoid duplicating the code in a program. It facilitates efficiency both in keep the code readible and short but also in facilitating its maintainence in the future.

It is a concept driven by the need to branch within a code i.e. to create code that a program runs should a condition be true or a condition be false.

What are the options?

There are many ways to iterate or "loop" within Code. There are many reasons driving the choice of loop, the first being whether the number of times required to loop is known or not.

  • For loop - when a condition is checked to see if it is true and the number of times needed to iterate is known.
  • Times loop - this is a compact For loop, again useful if number of iterations is known.
  • While loop - the code keeps running whilst the while condition is true, without needing to know the number of iterations.
  • Until loop - this is the reverse of the While loop, the code keeps running until the until condition is true.
  • Each loop - this instructs the code to iterate through each item in an array, checking the method condition against each item.

The above loops all return a copy of an element, elements or values from an array whilst maintaining the original array. The following return a new array.

  • Select loop - this is also an array but returns a new array, which is subset of an array.
  • Map loop - returns a transformed value by applying a function to each element, e.g. transforming 70% to 0.70.
  • Fold/Inject/Reduce loop - this reduces an array to a single value, that doesn't match the values in the array, e.g. the sum of an array.

Summary

Looping is a dynamic and efficient way to re-use code within a program. The type of loop is determined by the data structure in place, the activity and outcome required and the level of performance desired.

Aha: Javascript and Ruby are not an and/or concept. Both deliver functionality, ease of use and performance in different ways.
